# 企业微信营销应用建设项目 — 角色功能点需求分析文档

> **文档版本**：V1.0  
> **编制日期**：2026年6月16日  
> **文档性质**：基于《企业微信营销应用建设项目业务及技术需求-最终版》及角色总览图梳理的需求分析文档

---

## 一、文档概述

### 1.1 编写目的

本文档基于《企业微信营销应用建设项目业务及技术需求-最终版》及角色总览图片，对系统涉及的三类核心角色——**管理员**、**客户经理**、**客户**——进行系统化的功能点需求梳理与分析。旨在明确各角色在系统中的职责边界、操作权限、功能需求及交互关系，为后续系统设计、开发与测试提供清晰的需求基线。

### 1.2 系统定位

本项目旨在建设基于企业微信的客户经理私域营销工作台，打通行内现有营销系统，链接现有渠道，实现目标客户洞察、活动及产品批量触达及营销成效监测。系统核心能力包括：引流获客、客户认证、客户管理、客户运营，支持企业微信营销，完善客户行为标签，具备通过私域进行产品和服务推荐、活动分享、海报快搭、话术推荐、商机推送、营销SOP定制等客户经营能力。

### 1.3 角色总览

根据角色总览图，系统涉及三类核心角色，其终端形态与权限范围如下：

| 角色         | 终端形态                                             | 权限范围                                                                       | 角色定位                                                         |
| ------------ | ---------------------------------------------------- | ------------------------------------------------------------------------------ | ---------------------------------------------------------------- |
| **管理员**   | PC管理后台（B/S架构）                                | 全行/本机构（按机构层级隔离），总行管理员看全行，分行/支行管理员看本机构及下级 | 运营管理中枢，负责策略制定、任务下发、风控监测与数据看板         |
| **客户经理** | 企微工作台自建H5（手机端）+ PC端（侧边栏展示自建H5） | 名下的客户                                                                     | 一线营销执行者，系统核心使用者，负责客户经营、商机跟进、营销触达 |
| **客户**     | 微信端 H5                                            | 仅自己的信息                                                                   | 外部用户，被动接收营销内容与主动浏览产品/活动                    |

## 二、管理员角色需求分析

### 2.1 角色概述

管理员是系统的运营管理中枢，通过PC管理后台（B/S架构）进行操作。根据机构层级实现数据隔离——总行管理员可查看全行数据，分行/支行管理员仅可查看本机构及下级数据。管理员的核心职责包括：六库素材配置、标签与客群管理、场景与任务管理、风控合规监测、通讯录与角色管理、欢迎语配置、营销活动管理、统计分析与报表查看等。

### 2.2 六库配置

#### 2.2.1 功能描述

管理员负责维护系统"六库"（产品库、海报库、内容库、话术库、活动库、工具库）中的全部素材，包括素材的上下架、新增、删除、可使用机构及权限管理等操作。六库是客户经理开展营销活动的核心内容支撑，管理员的配置质量直接影响一线营销效果。

#### 2.2.2 功能细项

| 功能细项       | 需求描述                                                                                                                 |
| -------------- | ------------------------------------------------------------------------------------------------------------------------ |
| 素材上下架     | 管理员可对六库中的各类素材进行上架和下架操作，控制素材的可用状态，确保客户经理使用的素材始终为最新、合规的版本           |
| 素材新增       | 支持管理员新增各类素材，包括产品信息、海报图片、资讯文章、话术模板、活动配置、工具链接等，新增时需填写完整的素材属性信息 |
| 素材删除       | 支持管理员删除无效或过期的素材，删除操作需有确认机制，防止误删                                                           |
| 机构及权限管理 | 支持按机构维度配置素材的可见性和使用权限，不同层级的机构可使用不同的素材集合，实现精细化内容管控                         |

### 2.3 标签管理

#### 2.3.1 功能描述

管理员负责统一管理标签规则，包括预设标签的创建与维护、自定义标签的审核与管理、以及标签与客群管理的联动配置。标签体系是客户精细化运营的基础，直接支撑客户分群、精准营销和个性化推荐等核心业务场景。

#### 2.3.2 功能细项

| 功能细项     | 需求描述                                                                                               |
| ------------ | ------------------------------------------------------------------------------------------------------ |
| 统一标签规则 | 制定全行统一的标签分类规则和命名规范，确保标签体系的一致性和可扩展性，避免各机构自行定义导致的数据混乱 |
| 预设标签     | 管理员可创建系统预设标签，覆盖客户等级、资产区间、产品偏好、生命周期阶段等维度，供客户经理直接选用     |
| 自定义标签   | 支持客户经理申请自定义标签，管理员审核后生效，确保自定义标签的合理性和规范性                           |
| 联动客群管理 | 标签与客群管理联动，支持基于标签组合自动生成客群，实现标签驱动的精准客群划分                           |

### 2.4 客群管理

#### 2.4.1 功能描述

管理员通过360度筛选条件组合，批量筛选目标客户并添加标签，实现客群的精细化划分。客群管理是营销策略落地的前提，管理员需根据业务需求灵活定义客群规则，支撑后续的精准营销和差异化运营。

#### 2.4.2 功能细项

| 功能细项     | 需求描述                                                                                                   |
| ------------ | ---------------------------------------------------------------------------------------------------------- |
| 360筛选      | 提供多维度筛选条件（客户属性、行为数据、资产信息、标签组合等），支持条件的灵活组合与叠加，实现精准客群圈选 |
| 批量添加标签 | 对筛选出的客群批量添加标签，提升客群管理效率，支持一键标记和批量操作                                       |

### 2.5 场景管理

#### 2.5.1 功能描述

管理员根据业务场景规则生成添客二维码，客户扫码添加后自动打上对应标签，实现场景化的精准获客与自动标记。场景管理将线下触点（网点台牌、柜面、厅堂机具等）与线上引流打通，是引流获客的重要配置入口。

#### 2.5.2 功能细项

| 功能细项       | 需求描述                                                                     |
| -------------- | ---------------------------------------------------------------------------- |
| 场景规则配置   | 定义不同获客场景的规则，包括渠道来源、客户类型、自动标签等参数               |
| 添客二维码生成 | 根据场景规则自动生成带参数的二维码，扫码添加后系统自动完成标签标记和客户分配 |

### 2.6 任务管理

#### 2.6.1 功能描述

管理员负责创建、下发和跟踪各类业务任务（包括添客任务、营销任务等），并对任务执行情况进行可视化监控和排名统计。任务管理是连接管理层策略意图与一线执行落地的核心桥梁，确保营销策略能够高效、规范地传递至客户经理并得到执行。

#### 2.6.2 功能细项

| 功能细项 | 需求描述                                                                                                                   |
| -------- | -------------------------------------------------------------------------------------------------------------------------- |
| 任务下发 | 管理员可创建业务、添客、营销等任务模板，灵活分配至指定员工或团队，设置任务截止时间和执行要求，对执行进度进行全程可视化跟踪 |
| 任务排行 | 展示各员工/团队的任务完成情况排名，激励执行积极性，支持按机构、时间等维度查看                                              |
| 运营中心 | 通过关键指标可视化与多维度分析，实时掌握客户增长动态与团队业务表现，包含任务下发和执行情况的综合看板                       |

### 2.7 风控监测

#### 2.7.1 功能描述

管理员负责配置和监控风控合规相关功能，包括敏感词过滤、会话存档、违规提醒、超时回复监控等，确保客户经理的营销行为符合监管要求和行内合规标准。风控监测是银行类应用的核心合规保障模块，直接关系到监管合规和声誉风险防控。

#### 2.7.2 功能细项

| 功能细项     | 需求描述                                                                                                                                                     |
| ------------ | ------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| 敏感词过滤   | 可初始化和自定义敏感词库，制定敏感词过滤规则，在交流和触客过程中进行灵活、智能地敏感词过滤，保证营销服务的安全合规；可查看员工和客户触发敏感词记录，支持导出 |
| 会话存档     | 对客户经理与客户聊天沟通情况进行存档留痕处理，支持对聊天记录进行自动化解析和自主查询，可通过日期、关键词、员工、机构、敏感词等条件进行筛选                   |
| 违规提醒     | 针对单聊和群聊设置违规提醒规则，员工及客户触发敏感词时，系统立即给相应成员发消息提醒，提示可能出现的违规行为                                                 |
| 超时回复提醒 | 通过设置可自定义的超时规则，自动监控员工与客户单聊及群聊的响应时效，一旦会话超时未回复，系统立即向管理者发送预警，避免商机流失                               |
| 超时记录明细 | 完整记录所有会话超时的详细数据，提供可追溯、可分析的响应时效看板，通过可视化展示每一条超时记录详情，精准定位服务瓶颈，持续优化响应体系                       |

### 2.8 通讯录管理

#### 2.8.1 功能描述

管理员负责同步企业微信组织架构与成员信息，构建统一的团队管理视图，并对成员信息进行精细化维护与权限配置。通讯录管理是企业数字化运营的组织基础，需与人力资源系统/CRM系统保持数据同步。

#### 2.8.2 功能细项

| 功能细项     | 需求描述                                                                                                   |
| ------------ | ---------------------------------------------------------------------------------------------------------- |
| 企业成员管理 | 同步企业微信组织架构与成员信息，支持对成员信息进行精细化维护与权限配置，为企业数字化运营提供坚实的组织基础 |
| 角色管理     | 支持企业根据组织架构和业务需求，灵活创建并配置不同的权限角色，实现功能权限的精细化管理                     |

### 2.9 欢迎语配置

#### 2.9.1 功能描述

管理员配置包含文字、图片、图文链接与小程序的个性化欢迎素材，当客户添加员工企微后自动触发发送。欢迎语是客户接触企业的第一印象，直接影响客户对银行服务的感知和后续互动意愿。

#### 2.9.2 功能细项

| 功能细项       | 需求描述                                                                                 |
| -------------- | ---------------------------------------------------------------------------------------- |
| 欢迎语素材配置 | 支持配置包含文字、图片、图文链接与小程序的个性化欢迎素材，客户添加员工企微后自动触发发送 |

### 2.10 引流获客管理

#### 2.10.1 功能描述

管理员负责公海池客户名单的导入、分配与回收管理，以及管户分配规则的配置。引流获客管理是客户资源流转的核心管控模块，确保客户资源得到及时、有效的跟进，避免资源浪费，最大化销售机会。

#### 2.10.2 功能细项

| 功能细项             | 需求描述                                                                                                                                                                                       |
| -------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| 公海池客户名单导入   | 支持从公海池导入客户名单，选择名单后新建客户经理加好友任务，客户经理手机收到加好友任务通知后可手动执行加好友任务；也支持自动加好友任务，自动执行添加好友动作并给客户推送添加好友的服务通知     |
| 公海池未通过客户管理 | 通过公海池导入的客户信息，可下发自动加好友任务发送给客户经理或通过自动执行，任务执行成功后系统调用企业微信接口查看加好友情况，显示所有"执行状态"为"已执行"且"添加状态"为"失败"的加客户明细数据 |
| 公海池流失客户管理   | 显示客户删除客户经理或客户经理删除了客户的流失客户数据，记录客户流失的时间和客户流失的方式                                                                                                     |
| 公海池流失客户提醒   | 当有客户删除客户经理企业微信，发送消息提醒客户经理，同时在消息列表中展示历史的客户流失提醒消息                                                                                                 |
| 公海池客户回收       | 可自定义客户回收规则（如超N天未跟进、标记为无效客户），系统自动执行客户回收并将其释放至公共客户池，客户经理可从公海中主动申领或由管理员手动分配客户                                            |
| 管户分配             | 实现根据CRM现有管户关系完成各种场景添加外部联系人时的客户经理匹配，并可根据客户管户关系进行动态更新                                                                                            |

### 2.11 营销管理

#### 2.11.1 功能描述

管理员负责营销活动的创建与管理，包括客户群发任务的配置与下发、企微营销活动任务管理、SOP任务模板配置、产品管理、红包活动管理等。营销管理是管理员驱动一线营销执行的核心模块，通过标准化的任务配置和流程管理，实现营销策略的规模化落地。

#### 2.11.2 功能细项

| 功能细项                        | 需求描述                                                                                                                                                                                                                                                                                                                                 |
| ------------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| 客户群发-群发消息到客户手动执行 | 管理员筛选群发客户和执行人，选择H5活动、上传图片、编辑文本消息，支持手动发送和定时发送；群发任务下发到员工企微后，员工需手动点击发送按钮发送给客户                                                                                                                                                                                       |
| 客户群发-群发消息到客户自动执行 | 管理员筛选群发客户和执行人，配置消息内容，支持手动发送和定时发送；群发任务下发到员工企微后自动发送给客户                                                                                                                                                                                                                                 |
| 客户群发-管理员发表到客户朋友圈 | 管理员筛选群发客户和执行人，配置朋友圈文案和H5活动，支持手动发送和定时发送；朋友圈任务下发到员工企微后，员工需点击发送按钮发表到企业客户朋友圈                                                                                                                                                                                           |
| 客户群发任务数据监控            | 群发任务发布后，支持在监控台查看任务执行人数、客户人数、已完成/未完成任务人数，支持从机构执行明细、员工执行明细、客户明细三个维度查看任务数据                                                                                                                                                                                            |
| 企微营销活动任务                | 管理员创建企微营销活动任务，任务发布后客户经理企微接收营销助手任务消息提醒，查看并执行任务待办                                                                                                                                                                                                                                           |
| 任务数据监控                    | 任务发布后，支持在监控台查看任务执行人数、客户人数、已完成/未完成任务人数，支持从机构执行明细、员工执行明细查看任务数据                                                                                                                                                                                                                  |
| 任务模板管理                    | 支持添加一个或多个任务待办，填写待办名称、待办说明、场景话术                                                                                                                                                                                                                                                                             |
| SOP任务                         | 支持将标准化流程化的运营内容配置成SOP模板，通过配置SOP计划，选择任务生效日期、任务执行对象、筛选群聊/客户，到达设置的任务时间点自动推送对应的运营内容                                                                                                                                                                                    |
| 产品管理                        | 支持全行个人产品树的维护及相应可售产品的手机银行、微信银行购买链接配置，包含存款、理财、基金、国债、个贷、信用卡、分期等产品                                                                                                                                                                                                             |
| 红包管理                        | 支持配置红包发放规则（有效期、红包类型、活动资金、单个红包限额、领取人数、每人领取次数），根据规则创建红包活动，支持配置活动背景图、主题色、认证资料；红包活动通过二维码扫码参与获取客户信息，通过渠道活码在指定时段内吸引新客添加企业成员，系统借助欢迎语自动触发红包领取页面；客户首拆得奖即时到账，分享好友帮拆可让双方均获得现金奖励 |

### 2.12 统计分析与报表

#### 2.12.1 功能描述

管理员通过报表及看板模块，对全行/本机构的经营成效、客户增长、活动效果、合规情况等进行多维度统计分析。统计分析是管理员掌握业务全局、制定运营策略的数据基础，需支持按总-分-支-网层级汇总统计和下钻分析。

#### 2.12.2 功能细项

| 功能细项         | 需求描述                                                                                                                         |
| ---------------- | -------------------------------------------------------------------------------------------------------------------------------- |
| 经营成效看板     | 按照总-分-支-网层级汇总统计企微连接数、客户信任等级（互动频率、购买产品频次）、AUM提升等指标，支持按机构层级查看和下载           |
| 流失客户明细     | 记录流失客户明细、流失客户详细信息、客户流失时间及流失方式，支持按客户名称、客户手机号等查看客户流水记录                         |
| 活码统计         | 提供全方位的渠道效果分析能力，通过可视化数据看板和多维度统计指标，助力企业精准评估投放效果，优化营销策略                         |
| 活动统计         | 以单场活动为维度，统计浏览次数、浏览人数、拉新人数、查看产品页面人数、新增企微客户数量，支持客户数据埋点                         |
| 综合活动看板     | 以全部活动为维度，统计浏览次数、浏览人数、拉新人数、查看产品页面人数、新增企微客户数量，支持客户数据埋点                         |
| 客户经理客户报表 | 统计客户经理的客户增长情况，支持按客户经理维度和时间维度（天/周/月）查看客户经理名下的客户数据                                   |

## 四、客户角色需求分析

### 4.1 角色概述

客户是系统的外部用户，通过微信端H5页面与系统交互，权限范围仅限于自己的信息。客户不直接操作系统功能，而是被动接收客户经理推送的营销内容，或主动浏览产品、参与活动、完成认证等。客户体验的优劣直接影响营销转化率和客户满意度，因此客户侧的功能设计需以简洁、便捷、安全为核心原则。

### 4.2 客户认证

#### 4.2.1 功能描述

客户可通过客户经理转发的认证流程主动填写信息完成身份认证，认证多以活动形式呈现，降低客户操作门槛。认证完成后，系统自动建立行内客户身份与企微外部联系人的映射关系，为客户后续享受个性化服务奠定基础。

#### 4.2.2 功能细项

| 功能细项     | 需求描述                                                                                     |
| ------------ | -------------------------------------------------------------------------------------------- |
| 客户主动认证 | 客户在客户经理转发的认证流程中填写信息认证（多以活动形式），完成行内客户身份与企微身份的绑定 |

### 4.3 金融小店浏览

#### 4.3.1 功能描述

客户通过客户经理分享的金融小店链接，可浏览产品货架、了解服务内容、查看客户经理信息。金融小店是客户了解银行产品和服务的线上窗口，需支持跳转至手机银行完成购买操作。

#### 4.3.2 功能细项

| 功能细项         | 需求描述                                                           |
| ---------------- | ------------------------------------------------------------------ |
| 产品浏览         | 客户可在金融小店中浏览基金、存款、理财、信用卡、分期等分类产品信息 |
| 购买跳转         | 产品分享链接支持跳转至行方手机银行或微银行完成购买操作             |
| 客户经理信息查看 | 客户可查看客户经理的基本信息和联系方式                             |

### 4.4 活动参与

#### 4.4.1 功能描述

客户可参与客户经理或管理员推送的各类营销活动，包括红包活动、产品活动、节日活动等。活动参与是客户与银行互动的重要触点，需确保活动流程简洁、体验流畅。

#### 4.4.2 功能细项

| 功能细项       | 需求描述                                                                             |
| -------------- | ------------------------------------------------------------------------------------ |
| 红包活动参与   | 客户通过二维码扫码参与红包活动，首拆得奖即时到账，分享好友帮拆可让双方均获得现金奖励 |
| 活动浏览与报名 | 客户可浏览活动详情并完成报名、达标、领奖等操作                                       |
| 权益领取       | 客户经理派发的权益，客户可在企微聊天中或加好友时领取                                 |

### 4.5 内容浏览与互动

#### 4.5.1 功能描述

客户可浏览客户经理推送的各类内容（资讯文章、海报、产品推荐等），并产生浏览行为数据供客户经理追踪分析。客户的浏览行为数据是客户经理了解客户兴趣偏好的重要依据。

#### 4.5.2 功能细项

| 功能细项     | 需求描述                                           |
| ------------ | -------------------------------------------------- |
| 资讯浏览     | 客户可浏览客户经理推送的各类资讯文章和海报内容     |
| 产品页面浏览 | 客户可查看产品详情页面，浏览行为被记录用于营销分析 |
| 内容转发     | 客户可转发内容给好友，转发行为被记录用于传播分析   |

### 4.6 业务办理

#### 4.6.1 功能描述

客户通过客户经理分享的链接，可跳转至手机银行或微银行完成网点查询、排队取号、预约兑换现金、在线申卡等业务办理操作。业务办理功能将企微触点与行方核心渠道打通，实现从营销到业务转化的闭环。

#### 4.6.2 功能细项

| 功能细项     | 需求描述                                                                       |
| ------------ | ------------------------------------------------------------------------------ |
| 网点查询     | 客户点击客户经理分享的链接后可跳转至行方手机银行或微银行程序，自助查找网点信息 |
| 排队取号     | 客户点击客户经理分享的链接后可跳转至行方手机银行或微银行程序，自助进行排队取号 |
| 预约兑换现金 | 客户可通过客户经理分享的链接预约兑换现金                                       |
| 在线申卡     | 客户可预填申卡信息，提升业务办理效率，减少客户现场等候时间                     |

### 4.7 客服咨询

#### 4.7.1 功能描述

客户在客户经理非工作时间或其他必要情况下，可通过自动回复引导一键拨打客服热线（95507/95392），或跳转行方在线客服应用（含智能机器人、人工在线客服服务），确保客户服务不中断。

#### 4.7.2 功能细项

| 功能细项     | 需求描述                                                     |
| ------------ | ------------------------------------------------------------ |
| 客服热线拨打 | 客户可一键拨打95507/95392客服热线                            |
| 在线客服     | 客户可跳转行方在线客服应用，使用智能机器人或人工在线客服服务 |

## 六、非功能性需求摘要

### 6.1 关键技术要求

- 平台应具有充分的业务前瞻性、先进性和可扩展性，能够支持未来五年的业务发展需求
- 符合微服务分布式架构要求，数据模型设计合理具有前瞻性、支持灵活扩展
- 系统采用开放式架构，基于容器云或虚拟化环境部署，支持横向扩展
- 数据库使用招标人提供的数据库（GBASE、OceanBase）
- 须对接招标人统一的监控平台、系统日志收集管理及分析平台
- 所有对外发布和调用的服务接口必须按照招标人统一的报文规范进行改造，并接入招标人的服务总线平台
- 全面实施国家安全密码算法标准，系统内敏感数据、日志、报文等存储、传输、展示均需进行加密和脱敏处理
- 支持录入和存储生僻字，中文字集满足GB18030-2022要求，字符集编码使用UTF-8

### 6.2 性能要求

- 系统不应有客户数、并发量、法人数量、分支机构数量等许可限制
- 在预估30%年增长率条件下，保证满足未来5年发展需求
- 扩展能力应大于70%而且趋近于线性增长
- 要求中标人负责性能测试和调优工作

### 6.3 可用性要求

- 实现七×二十四小时的业务处理及稳定的系统运行
- 分布式高可用设计方案，要求实现单站点、单数据库等故障时可自动无缝转移接替服务
- 平均故障恢复时间（MTTR）小于半小时
- 上线试运行期内统计产品缺陷率低于0.32‰

### 6.4 安全性要求

- 安全机制与应用平台相对独立，使用招标人的加密方案
- 提供完备的、可查询的日志体系和切实可行的审计机制
- 系统对所有系统操作提供日志记录，各环节实施国密算法的保密性、完整性管理
- 客户端到服务端的访问必须采用https/sftp安全传输协议
- 应用系统的会话标识须保持随机且唯一，具备防数据重放机制和数据防篡改机制

### 6.5 接口规范要求

- 遵循招标人系统接口规范，支持多种渠道接入
- 系统对接接口符合招标人ESB服务规范标准及相关通讯协议
- 所有接口应由统一的API网关进行统一管控
- 涉及与腾讯开放平台等互联网区接口交互须提供准确的服务接口文档
